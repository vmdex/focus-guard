# Focus Guard Clock

## Версія

v0.1 - перший автономний Windows-застосунок у системі Focus Guard.

## Основна ідея

Focus Guard Clock - це простий застосунок для focus sessions, який допомагає працювати і відпочивати без зайвого шуму, дратівливих подвійних сповіщень і втрати контролю над наступною дією.

Перший продукт не має бути великою productivity-системою. Його задача - швидко дати користь у щоденній роботі і стати першим полігоном для майбутньої логіки Focus Guard.

## Проблема

Стандартний Windows Clock має корисну ідею focus sessions, але його поведінка не повністю підходить:

- після завершення сесії з'являється більше сповіщень, ніж потрібно;
- сповіщення про завершення і початок паузи можуть відчуватися як інформаційний шум;
- користувач не має достатнього контролю над тим, що саме має статись після завершення сесії;
- застосунок не фокусується на питанні: "що мені робити далі?".

## Ціль v0.1

Зробити мінімальний Windows-застосунок, який дозволяє запускати focus session, завершувати її зрозумілим одним сигналом і допомагати користувачу перейти до наступної запланованої дії.

## Користувач

Перший користувач - людина, яка:

- працює за Windows PC;
- може надто довго сидіти без паузи;
- може втрачати час після коротких перерв;
- хоче мати простий контроль над робочими сесіями;
- не хоче, щоб застосунок сам ставав джерелом роздратування.

## Принципи v0.1

- Одне завершення - одне сповіщення.
- Ніяких зайвих автоматичних повідомлень.
- Користувач завжди бачить поточний стан.
- Після сесії застосунок має допомогти зрозуміти наступну дію.
- Мінімум налаштувань, але важливі речі мають бути контрольовані.
- Застосунок має бути корисним навіть без акаунта, backend або синхронізації.

## Основні стани

### Idle

Сесія не запущена.

Користувач може:

- задати тривалість роботи;
- задати тривалість паузи;
- вказати наступну дію;
- запустити focus session.

### Focus

Йде робоча сесія.

Користувач бачить:

- залишок часу;
- назву або текст поточної дії;
- кнопку pause;
- кнопку stop;
- кнопку finish early.

### Break

Йде пауза.

Користувач бачить:

- залишок часу паузи;
- що буде після паузи;
- кнопку skip break;
- кнопку stop.

### Completed

Сесія завершена.

Користувач бачить:

- що сесія завершена;
- що було зроблено;
- що заплановано далі;
- кнопки start next focus, start break, done.

## MVP-функції

### Таймер focus session

Користувач може:

- вибрати або ввести тривалість роботи;
- запустити сесію;
- поставити на паузу;
- продовжити;
- зупинити;
- завершити достроково.

### Пауза

Користувач може:

- задати тривалість паузи;
- запускати паузу після focus session вручну або автоматично;
- пропустити паузу;
- завершити паузу.

Автоматичний старт паузи має бути налаштуванням, а не єдиною можливою поведінкою.

### Одне сповіщення після завершення

Після завершення focus session застосунок має показати одне сповіщення.

Приклади тексту:

- "Focus session завершено"
- "Час зробити паузу"
- "Готово. Далі: [наступна дія]"

У v0.1 не має бути окремого другого сповіщення про старт паузи, якщо користувач явно цього не ввімкнув.

### Наступна дія

Перед запуском сесії користувач може вказати короткий текст:

- "доробити задачу";
- "відповісти на лист";
- "почати перерву";
- "перейти до наступної справи".

Цей текст показується:

- у головному екрані;
- під час сесії;
- у сповіщенні після завершення;
- на екрані completed.

### Історія дня

Застосунок має зберігати просту історію за поточний день:

- час початку;
- час завершення;
- тривалість;
- тип: focus або break;
- статус: completed, stopped, finished early;
- текст наступної дії, якщо був.

У v0.1 достатньо локального зберігання.

## Налаштування v0.1

Потрібні мінімальні налаштування:

- default focus duration;
- default break duration;
- auto-start break on/off;
- notification sound on/off;
- show notification on focus end on/off;
- keep window always on top on/off.

## Не входить у v0.1

У першій версії не робимо:

- акаунти;
- backend;
- синхронізацію;
- календар;
- складну систему задач;
- блокування сайтів;
- блокування застосунків;
- Android-зв'язок;
- Chrome extension;
- статистику за тиждень або місяць;
- AI-планування.

Ці речі важливі для майбутнього, але не потрібні для першої корисної версії.

## Майбутні ідеї

Після v0.1 можна додати:

- шаблони сесій;
- набори задач на день;
- повторювані цикли work/break;
- різні типи сповіщень;
- fullscreen reminder;
- mini floating timer;
- tray mode;
- статистику за день;
- експорт історії;
- інтеграцію з майбутнім backend.

## Відкриті питання

- Чи має пауза запускатися автоматично за замовчуванням?
- Чи потрібно показувати fullscreen екран після завершення сесії?
- Чи має застосунок жити в system tray?
- Чи треба робити mini mode поверх інших вікон?
- Яка технологія найкраща для першої Windows-версії?
